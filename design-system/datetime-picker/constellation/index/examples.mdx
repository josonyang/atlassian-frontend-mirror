## Default

By default, selecting the date field opens the calendar view. The current date text is bold, underlined, and highlighted blue.

The time field is used to select a time from the [select](/components/select) menu.

## Form

<Example Component={DateTimePickerForm} packageName="@atlaskit/datetime-picker" />

### Validation

This is how date time picker behaves within [forms](components/form/).

Validation displays an error message related to the restrictions of the date time picker.

When a user selects the date time picker area, the focus color changes to blue.

When validating date time pickers in real-time, message icons switch based on the message type. For example, helper text becomes an error message when the input content doesn't meet the criteria. Error and warning messages disappear when the criteria is met.

Keep helper text as short as possible. For complex information, provide a link to more information in a new browser tab. Use the [messaging guidelines](/content/messaging-guidelines) for more help.

## Disabled dates

If a certain date is not a valid selection, you may disable it in the calendar shown to users. This does not restrict the dates that a user may type, so validation is necessary.

## Internationalization

Date time picker supports internationalization through three props:

- `locale` affects language, format and, parsing.
- `weekStartDay` determines the first day of the week shown on the calendar.
- `aria-label` determines the label used for assistive technology for both the date picker field or the time picker field.

### Locale

Use `locale` to tailor UI copy to local audiences.

<Example Component={DateTimePickerLocale} packageName="@atlaskit/datetime-picker" />

### Week start day

Use `weekStartDay` to adjust which day of the week is shown first in the calendar. A value of `0` corresponds to Sunday (default), `1` to Monday, and so on.

<Example Component={DateTimePickerWeekStartDay} packageName="@atlaskit/datetime-picker" />

### Labels

Use the field component to provide a proper group label. Use the date time picker's `aria-label` attribute to localize the labels that assistive technologies will use when selecting the date picker and the time picker.

## Date and time formats

Date time picker supports customizing the format of dates and times. Formats are given as strings and use the syntax specified at Modern JavaScript Date Utility Library.

- `dateFormat` determines how dates are formatted.
- `timeFormat` determined how times are formatted.

Where possible use `locale` for date and time formatting, instead of a custom format. Date and time formats should be informed by the userâ€™s locale and the use case.

## Accessibility

If using fields with labels other than "Date" and "Time" for the date picker and time picker respectively, use `datePickerSelectProps` and `timePickerSelectProps` to pass the `aria-label` prop. Assistive technologies will announce these along with the label from the field component when activated.

Set the `inputId` for the `datePickerSelectProps` to the `id` from the `fieldProps` provided by the field component. This is required for proper association of the label.
